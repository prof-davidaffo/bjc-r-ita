Programación Funcional

En el idioma utilizado en el examen, la única forma de generar datos de lista es con comandos, acciones que cambian el contenido de una lista: Agregar. Insertar. Reemplazar. Eliminar. Adoptan este enfoque porque creen que es el estilo de programación más fácil de entender y también para incluir Scratch, en el que estos comandos son el único mecanismo proporcionado para trabajar con listas. Pero hay una forma mejor, llamada programación funcional, en la que nunca modifica una estructura de datos, sino que crea una nueva para contener los nuevos valores.

En un momento, los programadores "reales" veían la programación funcional como algo de interés solo para los profesores de informática. Pero hoy en día es una forma preferida de programar debido al hardware de procesamiento paralelo, ya sea con bastidores llenos de computadoras o con chips de procesador multinúcleo que incluyen cuatro u ocho procesadores dentro de un solo chip. La razón es que con varios procesos ejecutándose en paralelo, es posible que dos de ellos intenten modificar la misma variable al mismo tiempo, con malos resultados. Pero si el programa está escrito funcionalmente, nunca se modifica nada, por lo que eso no puede suceder. Hay lenguajes funcionales en los que no hay equivalentes a set o replace.
