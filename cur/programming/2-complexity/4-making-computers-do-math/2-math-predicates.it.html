Creare una libreria matematica

In questo progetto, creerai una libreria (una collezione) di strumenti matematici che si basa sui blocchi creati in Laboratorio 3 Pagina 2: Combinare predicati.
AAP-2.M.3, AAP-3.D.3

Usare le librerie semplifica lo sviluppo di nuovi programmi. Quando usi procedure che sai già funzionare correttamente, riduci il tempo necessario per programmare, il numero di bug possibili nel tuo codice e quanto del tuo progetto devi testare.

: Libreria software

Una libreria software è una collezione di procedure che possono essere usate nei programmi.

  1. Esporta i blocchi matematici dal tuo progetto U2L3-Predicates.

    Esportare i blocchi

    1. Apri il progetto Snap! da cui vuoi esportare i blocchi. (In questo caso, il tuo progetto U2L3-Predicates.)
    2. Scegli "Export blocks..." dal menu File di Snap! ().
    3. Seleziona solo i blocchi che vuoi esportare. (In questo caso vuoi , , e .)
      Puoi deselezionare i blocchi che non ti servono oppure fare clic con il tasto destro sullo sfondo, scegliere "none" e poi selezionare solo quelli che vuoi.
    4. Le librerie Snap! e i progetti Snap! vengono esportati entrambi come file XML.
      Fai clic su "OK." Verrà scaricato un file XML.
  2. Importa questi blocchi nel tuo progetto U2L4-MathLibrary, testali, risolvi eventuali problemi e salva.

    Importare i blocchi

    1. Apri il progetto Snap! in cui vuoi importare i blocchi. (In questo caso, il tuo progetto U2L4-MathLibrary.)
    2. Importa il file XML trascinando il file XML dalla cartella dei download nella finestra di Snap!.
      Puoi anche importare il file XML scegliendo "Import..." dal menu File di Snap!, trovando il file XML sul tuo computer e cliccando "Open".
    3. Testa i blocchi che hai importato:
      1. Trova i blocchi importati alla fine delle palette che li contengono.
      2. Clicca ciascun blocco importato per assicurarti che funzioni correttamente.
Stai creando una libreria matematica. Snap! ha librerie per molte cose, come una per accedere a diverse parti di parole e frasi...

Ora hai creato una piccola libreria, e potresti voler scrivere un elenco di istruzioni su come usare le funzioni della tua libreria (per esempio, divisors of prende un intero positivo come input e riporta una lista di numeri). Le istruzioni formano una Application Program Interface per la libreria.

: Application Program Interface (API)
AAP-3.D.4, AAP-3.D.5

Un'application program interface (API) documenta ciò che un programmatore deve sapere per usare una libreria: è una descrizione dello scopo di ogni procedura, dei suoi input e output (ma non dei suoi algoritmi).

Che cos'è una Web API?

Un tipo comune di API è una web API in cui la libreria esiste su un computer di qualcun altro. Per esempio, la Google Maps API descrive come incorporare una mappa di Google Maps nel tuo sito.

Una web API è solo una chiamata a una procedura su un'altra macchina. Per esempio, queste sono tre notazioni diverse per la stessa chiamata di procedura, che cerca il numero di ricerche per "BJC" negli Stati Uniti:

  • https://trends.google.com/trends/explore?q=BJC&geo=US 
  • EXPLORE (q = BJC, geo = US)

L'unica differenza è che l'URL mostra dove su Internet trovare la procedura.
