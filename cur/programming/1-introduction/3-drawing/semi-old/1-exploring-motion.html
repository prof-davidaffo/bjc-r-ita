: Algorithm and Pseudocode
(AAP-)2.A.1, AAP-2.A.2, AAP-2.A.3

An algorithm is a finite sequence of steps that are usually performed by a computer. The algorithm doesn't have to be written in any particular programming language or even in a programming language at all; you can write your algorithm in English or any other human language or using a diagram. Some people call an algorithm written in human language pseudocode. Once you know the steps that the computer will take, you can code your algorithm in the programming language of your choice.

Pseudocode can make wishful thinking too easy. You can write anything as a line of pseudocode: "Pick the winning lottery numbers" or "Modify Facebook to make 1,000 people like my post." Pseudocode only works if you have a clear understanding of what the computer can really do. And the only way to be sure of that is to code in the language you will be using. Pseudocode for a Snap! program will look different from pseudocode for a Java program.

    Snap! has a tool called Visible Stepping that allows you to control how quickly Snap! steps through the blocks of your code.

    • Click the Visible Stepping button (), and adjust the slider to control the stepping speed. The sprite slowly executes each step of the code while it is highlighted in cyan.
      Click for a video of adjusting the Visible Stepping slider.
    • You can slow the stepping to one just step at a time. Move the slider to the slowest setting (all the way to the left), and the shape of the yellow Run/Pause button will change (). Now, you can press Run/Pause button to single step through your code and see what happens at each step.
      Click for a video of single-stepping through code.

One way visible stepping is useful is to help you read someone else's code, by watching it unfold at human speed instead of computer speed. That's what you're doing here. Visible stepping is also useful in fixing problems with (debugging) your own code when it's doing something wrong and it's hard to see what.
